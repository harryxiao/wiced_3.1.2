wiced_result_t wiced_bt_deinit ( void  )

Deinitialise the WICED Bluetooth Framework.

This function tears down all active framework components. Depending on the hardware platform used, it may also power down the Bluetooth device.

Returns
WICED_SUCCESS : on success; WICED_ERROR : if an error occurred
wiced_result_t wiced_bt_init ( wiced_bt_mode_t  mode,
const char *  device_name 
)

Initialise the WICED Bluetooth Framework.

This function initialises the local Bluetooth device and the framework core components to operate in the mode specified. Upon return, the device is powered and discoverable.

Note
To switch mode, invoke wiced_bt_deinit to tear down the current operating mode, and call this function with the desired mode.
Parameters
mode: The framework mode of operation
device_name: A user-friendly name of the local Bluetooth device. A name longer than 21 characters will be truncated.
Returns
WICED_SUCCESS : on success; WICED_ERROR : if an error occurred
wiced_result_t wiced_bt_init_address ( const wiced_bt_device_address_t *  address,
const wiced_bt_device_address_t *  mask 
)

Initialise the device address of the local Bluetooth device.

This function provides users with a option to overwrite the default address of the local Bluetooth device with the address provided. Once called, wiced_bt_init() overwrites the default address with the address provided. Users can selectively overwrite bits of the default address by setting the correspondping bits in the 'mask' argument to 1.

Warning
When used, this function MUST be called before wiced_bt_init()
Parameters
[in]address: new address
[in]mask: masking bits
Returns
WICED_SUCCESS : on success; WICED_ERROR : if an error occurred
