wiced_result_t wiced_bt_rfcomm_connect ( wiced_bt_rfcomm_socket_t *  socket,
const char *  service_name,
const wiced_bt_uuid_t *  uuid 
)

Connect to an RFCOMM server running on a remote Bluetooth device.

This function tells the local Bluetooth device to initiate an RFCOMM connection with a remote device. Once the connection is accepted, RFCOMM_EVENT_CLIENT_CONNECTED will be reported via the socket callback function provided.

Parameters
socket: A socket which will listen for an incoming connection
service_name: Service name of the RFCOMM server to connect to
uuid: UUID of RFCOMM server to connect to
Returns
WICED_SUCCESS : on success; WICED_BADARG : if bad argument(s) are inserted; WICED_PENDING : if socket already in connecting state; WICED_UNSUPPORTED : if mode of operation is not supported; WICED_ERROR : if an error occurred.

wiced_result_t wiced_bt_rfcomm_disconnect ( wiced_bt_rfcomm_socket_t *  socket)

Close an RFCOMM connection.

If in connected state, this function tells the local Bluetooth adapter to disconnect the RFCOMM connection and de-initialise the socket. If the socket is still in listening state, it will stop listening before deinitialising the socket.

Parameters
socket: The socket which will have the connection closed
Returns
WICED_SUCCESS : on success; WICED_ERROR : if an error occurred.
wiced_result_t wiced_bt_rfcomm_init_socket ( wiced_bt_rfcomm_socket_t *  socket,
wiced_bt_rfcomm_event_handler_t  handler,
void *  arg 
)

Initialise a WICED Bluetooth RFCOMM socket.

This function initialises an RFCOMM socket

Parameters
socket: A pointer to the socket to initialise
handler: A callback function provided by the user that is invoked when any of the wiced_bt_rfcomm_event_t events occurs. A valid callback function must be provided.
arg: An argument provided by the user which will be passed to the callback function
Warning
Only Packet, RFCOMM Packet Comms and RFCOMM Buffer Comms function calls are are allowed from the callback function.
Returns
WICED_SUCCESS : on success; WICED_BADARG : if bad argument(s) are inserted; WICED_ERROR : if any other error occurred.
wiced_result_t wiced_bt_rfcomm_listen ( wiced_bt_rfcomm_socket_t *  socket,
wiced_bool_t  auto_accept,
const char *  service_name,
const wiced_bt_uuid_t *  uuid 
)

Listen for an RFCOMM connection from a remote Bluetooth device.

This function tells the local Bluetooth device to listen for an incoming RFCOMM connection from a remote device.

Warning
: Only auto-accept is supported currently. Once the connection is accepted, RFCOMM_EVENT_CONNECTION_ACCEPTED will be reported via the callback function provided.
Parameters
socket: A socket which will listen for an incoming connection
auto_accept: Accept the connection automatically if equals WICED_TRUE
service_name: Service name of the RFCOMM connection's SDP record
uuid: UUID for of RFCOMM connection's SDP record
Returns
WICED_SUCCESS : on success; WICED_BADARG : if bad argument(s) are inserted; WICED_PENDING : if socket already in listening state; WICED_UNSUPPORTED : if mode of operation is not supported; WICED_ERROR : if an error occurred.
